# SEO UX Guardrails

Этот документ фиксирует ограничения для SEO-доработок, чтобы не ломать app-like UX ленты.

## Критичные зоны (не менять без отдельного approval)

- `src/components/feed/use-feed-page.ts`
- `src/components/feed/FeedScrollLinkCapture.tsx`
- `src/app/layout.tsx` (часть со `scrollRestoration`)
- `src/components/feed/PostCard.tsx` (механика overlay/click-through)
- `src/app/api/feed/route.ts` (контракт клиентской ленты)

## Что обязательно проверять после SEO-изменений

1. Переход `/ -> /p/[slug] -> назад` возвращает на ту же позицию списка.
2. Категории на главной (`/?category=`) переключаются без full reload.
3. Кнопка `Показать ещё` и автоматическая догрузка продолжают работать.
4. Переходы между постами остаются клиентскими и без дерганий скролла.
5. Главная и страницы постов визуально не меняют привычную структуру.

## Безопасный принцип внедрения

- SEO-роуты (`/category/*`, `/archive`) внедряем как **параллельный слой**.
- Не подменяем существующую ленту серверной пагинацией.
- Метаданные и canonical управляем в серверных `generateMetadata`.
- Усиление перелинковки делаем через дополнительные текстовые ссылки, а не заменой текущей навигации.
