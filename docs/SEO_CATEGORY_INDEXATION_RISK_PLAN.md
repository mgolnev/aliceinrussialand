# SEO: план по категориям с контролем UX-риска

## Цель
Увеличить SEO-охват по тематическим категориям без вмешательства в текущую механику ленты, `back`-навигацию и `scroll restoration`.

## Текущее безопасное состояние
- Фильтрация категорий работает через `?category=...`.
- Каноникал главной страницы консолидирован в `/`.
- Логика клиентской навигации и восстановления позиции скролла стабильна.

## Рискованный вариант (не внедрять без пилота)
- Делать категории индексируемыми отдельными SEO-страницами (уникальные `title/description/canonical` для каждой категории или отдельные роуты).
- Потенциальный SEO-эффект: высокий для long-tail запросов.
- **Риск затронуть UX-навигацию: да**.
- Уровень риска: `high` (может повлиять на историю переходов, ожидания от кнопки «назад», поведение фильтров и стабильность ленты).

## Безопасная альтернатива (основная)
- Сохранить текущую стратегию canonical-консолидации главной страницы.
- Усиливать discoverability через:
  - `sitemap.xml` (все посты),
  - внутренние ссылки с карточек/блока «Смотреть дальше»,
  - структурированные данные (`WebSite`, `Article`).

## Условия для пилота risky-варианта
1. Отдельная фича-флаг стратегия для индексации категорий.
2. A/B или staged rollout на части категорий.
3. Обязательные guardrails:
   - не менять механизм `replace`/`back`,
   - не менять текущий `scrollRestoration`,
   - не менять клиентский флоу догрузки ленты.
4. Мониторинг после запуска:
   - GSC: индексируемость категорий, canonical selection,
   - UX: возврат назад, глубина скролла, ошибки навигации.
